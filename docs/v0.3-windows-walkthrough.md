# v0.3 Windows 実機検証 walkthrough

> **対象**: 田中さん（Windows 実機所有）
> **目的**: v0.3 で Windows Credential Manager 経由のパスワード保存・取得・削除が動くことを確認し、tag を切る前の最終ゲートにする
> **想定所要時間**: 15〜20 分
> **要件**: Windows 10 / 11 + PowerShell 5.1 以上（Windows 10 にデフォルトで入っている）+ Node.js 22 以上

---

## 事前準備（5 分）

PowerShell（管理者権限不要）で：

```powershell
# Node 22+ が入っているか確認
node --version
# pnpm が入っているか確認（なければ corepack enable で有効化）
pnpm --version

# リポジトリを clone or pull
cd $env:USERPROFILE\Projects
git clone https://github.com/aiftp-tools/aiftp.git aiftp-v0.3-test
# あるいは既存のクローンがあれば
# cd aiftp-v0.3-test && git fetch origin && git checkout v0.3-dev

cd aiftp-v0.3-test
git checkout v0.3-dev
pnpm install
pnpm -r build
```

確認：

```powershell
node .\packages\cli\dist\bin.js --version
# 0.0.0 と出れば OK（npm 配布前なので version は固定）
```

エイリアスを張る（PowerShell プロファイルに足してもよい）：

```powershell
Set-Alias aiftp "$pwd\packages\cli\dist\bin.js"
# あるいは
function aiftp { node "$env:USERPROFILE\Projects\aiftp-v0.3-test\packages\cli\dist\bin.js" $args }
```

## Step 1: テスト用ディレクトリで `aiftp init`（5 分）

実 FTP サーバには接続しないので、host はダミーで OK。Credential Manager にエントリが入るかだけを確認します。

```powershell
mkdir aiftp-windows-test
cd aiftp-windows-test
echo "node_modules/" > .gitignore

aiftp init
```

対話で以下を入力（FTP サーバは実在しなくて構いません）：

```
Profile name: production
FTP host: test.example.local
FTP port: 21
Protocol: FTPS
FTP user: testuser
Remote root: /
Local root: .
Keychain service: aiftp:windows-test
Server kind: Generic
FTP password: any-password-you-like-for-testing
Store encrypted backups locally? Yes
```

期待される最終出力：

```
Initialized aiftp profile production
```

Credential Manager に **2 つのエントリ**ができているはず（FTP パスワード + バックアップ暗号鍵）：

```powershell
cmdkey /list:aiftp:windows-test:testuser
# 期待: Generic 種類のエントリが 1 つ表示される
cmdkey /list:aiftp:windows-test:backup-key:production
# 期待: もう 1 つ表示される
```

---

## Step 2: パスワードを read で取得できるか確認（3 分）

```powershell
aiftp doctor --profile production --json | ConvertFrom-Json | Select-Object -ExpandProperty summary
```

期待される値：

```
pass  : 4    # config-file / profile-exists / gitignore / keychain
warn  : 0 or 1
fail  : 2 or 3  # dns/tcp は失敗 (test.example.local が解決できない) -- 想定内
skip  : 6    # FTPS 系（接続できないから skip）
```

**ここで `keychain: pass` が出ていれば、Windows Credential Manager の get_password が動いている = v0.3 の主目的達成**です。

詳細を見たい場合：

```powershell
aiftp doctor --profile production --json | ConvertFrom-Json | Select-Object -ExpandProperty results | Where-Object id -eq "keychain"
```

---

## Step 3: `aiftp auth list` / `aiftp auth set` / `aiftp auth delete`（3 分）

各 auth コマンドが Credential Manager と正しく通信できるか：

```powershell
aiftp auth list
# 期待: production testuser stored

aiftp auth set --profile production
# 新しいパスワードを入力 → Updated credentials for production

cmdkey /list:aiftp:windows-test:testuser
# 期待: まだ 1 つ存在（上書きされた）

aiftp auth delete --profile production
# 期待: Deleted credentials for production

cmdkey /list:aiftp:windows-test:testuser
# 期待: 存在しない (エラー or 表示なし)

aiftp doctor --profile production --json | ConvertFrom-Json | Select-Object -ExpandProperty summary
# 期待: keychain が fail に変わる
```

---

## Step 4: クリーンアップ（2 分）

```powershell
# 残った backup-key を削除
cmdkey /delete:aiftp:windows-test:backup-key:production

# テストディレクトリごと削除
cd ..
Remove-Item -Recurse -Force aiftp-windows-test
```

## トラブルシューティング

### `Add-Type` でエラーが出る

PowerShell 5.1 以上なら標準で使えます。古い PowerShell（v4 以下）の場合は更新を：

```powershell
$PSVersionTable.PSVersion
# Major が 5 以上であること
```

### `cmdkey` が見つからない

cmdkey は Windows 10+ 標準ですが、`%PATH%` に `C:\Windows\System32` が含まれていない可能性。
PowerShell を再起動して試してください。

### Credential Manager に書き込めない

ドメインポリシーで Credential Manager が制限されている可能性。`gpedit.msc` で
"User Configuration → Administrative Templates → Windows Components → Credential User Interface"
を確認、または個人 PC でテストしてください。

### `node .\packages\cli\dist\bin.js` で「.js は実行できません」

PowerShell の実行ポリシーが厳しすぎる可能性：

```powershell
Get-ExecutionPolicy
# Restricted の場合は：
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```
