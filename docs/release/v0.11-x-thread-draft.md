**字数**: 約 130 字 + リンク 23 字

### Tweet 2 (問題提起)

```
Lolipop / Sakura / Xserver / StarServer 等、日本のレンタルサーバーは SSH 不可なプランが多い。

つまり rsync も git pull も WP-CLI も使えない。

AI agent (Claude Code / Codex) からは FTP を直接叩くしかなく、wp-config.php / .env の誤デプロイ事故リスクが常にある。
```
**字数**: 約 140 字

### Tweet 3 (アプローチ)

```
aiftp は「SSH/WP-CLI/GUI に頼らず、AI と安全にファイル deploy する」ニッチに振り切って設計。

・暗号化 default ON (AES-256-GCM)
・認証情報 hard-exclude (.env / wp-config.php)
・prepare→confirm の 2 段ゲート
・Keychain only (CLI 引数で渡さない)
```
**字数**: 約 140 字

### Tweet 4 (v0.11 新機能)

```
v0.11 新機能:

📦 WP テンプレ7種 (SWELL / Lightning / Cocoon / standard / static / Laravel / php-simple)
🔐 SFTP 対応 (SSH鍵 0o600/0o400 強制 + symlink 拒否)
↩️ PromptFlow :back 戻りナビ
🤖 MCP aiftp_init_template_list tool
🧪 3 OS × 2 Node smoke CI
```
**字数**: 約 140 字

### Tweet 5 (セキュリティ姿勢)

```
release 前に Codex に 3 ラウンドレビュー依頼:
1. テンプレ系 → HIGH 2 件 fix
2. SFTP 系 → CRITICAL 1 + HIGH 3 件 fix
3. セキュリティ専用 → HIGH 3 + MEDIUM 2 件 fix

特に 2 で「schema 上は鍵を受けるのに deploy 経路で password fallback」を発見。Codex なしでは出荷していた。
```
