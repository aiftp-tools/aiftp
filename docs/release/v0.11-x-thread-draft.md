```
aiftp v0.11.0 をリリースしました 🚀

できあがった HTML を毎回 FTP で手動アップする作業が面倒すぎて、AI と一緒にデプロイまで完結させたかった。

日本のレンタルサーバーで動く、AI 連携 FTP/FTPS/SFTP ツールです。

https://github.com/aiftp-tools/aiftp/releases/tag/v0.11.0
```
**字数**: 約 140 字 + リンク 23 字

### Tweet 2 (問題提起)

```
Lolipop / Sakura / Xserver / StarServer 等、日本の格安レンタルサーバーは SSH 不可なプランが多い。

rsync も git pull も WP-CLI も使えない。

非エンジニアの Web デザイナーが手動 FTP で頑張っているのを仕事柄よく見るので、ここに AI を入れたら一気に楽になるはず、と思いました。
```
**字数**: 約 140 字

### Tweet 3 (設計の柱)

```
「AI に FTP を直接叩かせる」だけだと .env や wp-config.php の誤デプロイ事故が起きる。

aiftp は「AI に安全に deploy させる」を中心に設計:

・暗号化 default ON (AES-256-GCM)
・認証情報の hard-exclude
・prepare→confirm の 2 段ゲート
・パスワードは Keychain のみ
```
**字数**: 約 140 字

🧪 3 OS × 2 Node smoke CI

## どちらを選ぶか — 判断材料

| 軸 | Pattern A (製品ドリブン) | Pattern B (観察ドリブン) |
|---|---|---|
| 読みやすさ | ◎ 製品が何かすぐ分かる | △ 1 ツイート目で製品名が出ない |
| 拡散しやすさ | △ 製品宣伝 RT は伸びにくい | ◎ 「AI は自分に甘い」観察は刺さる |
| ブランド形成 | △ ツール紹介ぽい | ◎ 「観察を共有する人」になれる |
| フォロワー外への到達 | △ 既存層中心 | ◎ 引用 RT で外に出る可能性 |
| 製品理解 | ◎ 明確 | ○ 段々わかる |

**推奨**: 田中さんの個人ブランド (「観察を言語化する PM」) に合うので **Pattern B 推奨**。ただし「製品が後段」なので、フォロワーが既に技術寄りなら Pattern A でも OK。

## 補足: 「AI 同士のクロスチェック」観察を独立ポストにする案

X スレッドに混ぜず、**別の独立ポスト** にしてもよい:

```
AI で開発していて気づいたこと:

AI は、自分自身の出力には甘い判断をする。モデルが Opus 4.7 や GPT-5.5 になっても、サブエージェントを使っても変わらない。

でも Claude に GPT を、GPT に Claude をレビューさせると、競合だからか厳しい。これが品質向上に効きました。
```

これは aiftp の文脈なしで成立する観察ツイートなので、製品 release スレッドとは別日に出すと、二段階で露出を確保できる戦略もあります。aiftp スレッドからの引用 RT として再利用も可能。
