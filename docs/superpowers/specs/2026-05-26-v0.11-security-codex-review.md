## Findings

### 1. HIGH / CWE-22, CWE-94: profile 名が config schema で未検証

**ファイル**: `packages/core/src/config.ts:150`, `packages/cli/src/index.ts:216`

`z.record(z.string(), profileSchema)` のため `[profile."../../../tmp/x"]` のような quoted TOML key を受け入れる。CLI init は `[profile.${answers.profile}]` を直接生成。`stateDir()` / `backupRoot()` は profile 名を path join に使うため、状態・バックアップ保存先の **traversal につながる**。

**攻撃シナリオ**: 悪意ある `.aiftp.toml` を含む repo を AI が扱い、MCP/CLI に同 profile を指定すると `.aiftp/state` / `.aiftp/backups` 境界を外れる。

**修正方針**: **release blocker**。`isValidProfileName()` を config load、init parse、import 生成名、MCP profile schema に適用。

### 2. HIGH / CWE-22: `remote_root` / FileZilla `RemoteDir` の `..` が未拒否

**ファイル**: `packages/core/src/config.ts:27`, `packages/core/src/importers/filezilla.ts:117`, `packages/core/src/deploy.ts:123`

`remote_root` は空文字以外ほぼ無検証。FileZilla import も decoded segment をそのまま `/${segments.join('/')}` にする。deploy は `posix.join(remoteRoot, path)` で upload/delete 先を作るため、`/public_html/../../other` が **remote root 外操作**になる。

**攻撃シナリオ**: 悪意ある XML/config を import し、後続 push/rollback/delete が想定外 remote path に作用。

**修正方針**: **release blocker**。remote path segment に `.` / `..` / slash / control char を拒否し、FileZilla decode 時も同じ正規化を通す。

### 3. HIGH / CWE-295: SFTP host key verification が未実装

**ファイル**: `packages/core/src/sftp-client.ts:182`

`SftpClient.buildConnectConfig()` は host/port/user/timeout だけで、known_hosts または TOFU pinning なし。

**攻撃シナリオ**: MITM が偽 SFTP server を提示し、password auth の資格情報や private-key signing oracle を悪用。

**修正方針**: **release blocker または明示的 limitation 付き延期**。最低でも初回 host key fingerprint 表示 + pin 保存、変更時 hard fail。延期なら README/doctor/init に「known_hosts 検証なし」を明記。

**v0.11 採用判断**: TOFU 実装は規模大 (`.aiftp/known_hosts` + diff prompt + first-connect UX) のため **v0.12 で実装、v0.11 は明示的 limitation 記載**。

### 4. MEDIUM / CWE-22, CWE-367: `ssh_key_path` の traversal / symlink / TOCTOU が未ハードニング

**ファイル**: `packages/core/src/path-utils.ts:27`, `packages/core/src/sftp-client.ts:136`

`expandTilde()` は `~/` を `join(homedir(), ...)` するだけで `~/../../../...` を拒否しない。key 読み込みは `statSync()` 後に `readFileSync()` するため symlink swap の TOCTOU も残る。

**攻撃シナリオ**: 悪意ある config が意図しない 0400/0600 local file を key として読ませる、または stat/read 間に差し替える。

**修正方針**: **release blocker 推奨**。`..` 拒否、`realpath` + `lstat` symlink 拒否、open handle + `fstat` 後 read に変更。

### 5. MEDIUM / CWE-200, CWE-732: MCP FileZilla import confirm が `.aiftp.toml` の 0o600 を落とす

**ファイル**: `packages/mcp/src/index.ts:1880`

MCP confirm は temp file を `writeFile(tmpPath, source, 'utf8')` で作って rename。**mode 指定がないため通常 umask で 0o644** になり、既存 0o600 config を world-readable に置換。

**攻撃シナリオ**: profile host/user/keychain_service/将来の ssh_key_path が他 local user に見える。

**修正方針**: **release blocker**。temp write は `{ encoding: 'utf8', mode: 0o600 }`、可能なら既存 mode を維持。

### 6. LOW / CWE-400: FileZilla XML parser が entity 処理を有効化

**ファイル**: `packages/core/src/importers/filezilla.ts:445`

`processEntities: true`。現在 lock は `fast-xml-parser@5.8.0` で Snyk 上は既知 direct vuln なし。過去の entity expansion CVE は `processEntities: false` が緩和策。

**修正方針**: **v0.11.1**。FileZilla import では entity 展開不要なので false、入力サイズ上限も追加。

### 7. LOW / CWE-664: SFTP upload が atomic rename ではない

**ファイル**: `packages/core/src/sftp-client.ts:226`

`upload()` / `uploadBuffer()` は remote path へ直接 `put()`。

**攻撃シナリオ**: 接続断で remote に partial file が残り、PHP/JS が壊れた状態で配信される。

**修正方針**: **v0.11.1**。`.aiftp-tmp-<uuid>` へ put 後 rename、失敗時 cleanup。
