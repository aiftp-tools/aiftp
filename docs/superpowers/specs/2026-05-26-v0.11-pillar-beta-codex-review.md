## Phase 1: 設計レベル

### 1. [HIGH] `safety.prod_profile_patterns` の default `main*` 消失

**ファイル**: `packages/core/src/config.ts:57`, `packages/core/src/templates/wordpress-swell.ts:26`, `packages/cli/src/index.ts:231`

テンプレ適用時に `prod_profile_patterns` が `["*prod*", "*www*", "*-live"]` へ置換され、schema default の `main*` が失われる。`main` プロファイル使用時に production type-to-confirm gate が外れる。

**修正方針**: 全テンプレの production patterns に `main*` を含める。`live*` / `honban*` も検討。

### 2. [MEDIUM] WP `debug.log` の hard-exclude が wordpress-standard のみ

**ファイル**: `wordpress-{swell,lightning,cocoon,standard}.ts`

`wp-content/debug.log` の除外が standard のみ。SWELL/Lightning/Cocoon に欠落。

**修正方針**: WP 共通 baseline を抽出し 4 テンプレすべてに適用。

### 3. [MEDIUM] `vendor/**` hard exclude がレンタルサーバー運用と矛盾

**ファイル**: `templates/laravel.ts:19`, `templates/php-simple.ts:20`, `core/src/exclude.ts:262`

SSH/Composer 不可のレンタルサーバーでは `vendor/` を FTP 配置するケースがあり、正当な deploy ファイルが除外される。

**修正方針**: `vendor/**` は hard exclude から通常 exclude/推奨コメントに落とすか、テンプレ別に方針分岐。

### 4. [INFO] EC-CUBE / Movable Type / CGI-Perl 系が未収録

v0.11 初版 release blocker ではないが、将来 backlog 候補。

## Phase 2: 実装レベル

### 1. [HIGH] `localRoot` が `template.defaults` で無条件上書き

**ファイル**: `packages/cli/src/init-flow.ts:105`, `packages/cli/src/index.ts:204`, `packages/cli/src/index.ts:403`

`static` / `laravel` で summary review で確認・編集した `localRoot` 値が、最終 render 時に template default で上書きされる。ユーザーが見た値と TOML に書かれる値がズレる。

**修正方針**: template 選択時に `localRoot` の initial 値に template default を反映し、summary も最終値を表示。ユーザー編集値を最優先。

### 2. [MEDIUM] preflight `php_lint` / `json_check` が runtime に未接続

**ファイル**: `packages/core/src/config.ts:133`, `packages/cli/src/index.ts:1721`, `packages/mcp/src/index.ts:682`

schema と TOML 出力には `[preflight]` が存在するが、CLI/MCP runtime は config を参照せず常に `checkAll(paths)` を渡す。テンプレの ON/OFF が実動作に効かない。

**修正方針**: `checkAll` に config 由来の有効/無効を渡すか、未対応なら `[preflight]` 出力を v0.11 では外す。

### 3. [LOW] MCP tool が `defaults` を返さず AI 利便性が下がる

**ファイル**: `packages/mcp/src/index.ts:1246`

`aiftp_init_template_list` は `id/description/longDescription` のみ。`defaults` は静的 preset で secret ではないため、AI が `localRoot` や hard-exclude 内容を判断できない。

**修正方針**: read-only な defaults summary を含めるか、別 tool `aiftp_init_template_show` を追加。

### 4. [LOW] テストの edge case 不足

**ファイル**: `packages/cli/src/index.spec.ts:212-264`

未カバー: 空文字 `--template`、`--template list` への余剰引数、template-select 後 cancel、static/laravel の `localRoot` round-trip。

### 5. [INFO] `TemplateConfigSchema.strict()` は妥当

未知フィールドを拒否する実装は正しい。問題なし。
