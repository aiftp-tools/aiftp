# Code of Conduct

aiftp follows a pragmatic adaptation of the
[Contributor Covenant 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).
We've kept it short and aligned with how the project actually
operates today.

---

## Our pledge

We — maintainers and contributors — commit to making aiftp a
welcoming and harassment-free environment for everyone, regardless
of age, body size, visible or invisible disability, ethnicity, sex
characteristics, gender identity and expression, level of experience,
education, socio-economic status, nationality, personal appearance,
race, religion, or sexual identity and orientation.

We commit to interacting in ways that build a healthy community.

## Expected behavior

- Be considerate and respectful in code review, issues, and
  discussions.
- Disagreement on technical decisions is welcome; personal attacks
  are not.
- Assume good intent when reading others' messages, especially across
  language barriers (this is a Japanese-led, English-default project —
  many participants are writing in their second language).
- Give credit where due. Use first-person plural ("we") when
  describing collective decisions in issues and PRs.
- When closing an issue or rejecting a PR, explain the reason
  briefly and respectfully.

## Unacceptable behavior

- Harassment, insults, demeaning jokes, or personal attacks.
- Public or private harassment, including unwanted sexual attention.
- Publishing others' private information without their explicit
  permission (e.g. real-world identity, employer, hosting
  credentials).
- Repeated derailing of technical discussions with off-topic content.
- Other conduct which could reasonably be considered inappropriate
  in a professional setting.

## Scope

This Code of Conduct applies to all project spaces — issues,
discussions, pull requests, the project's GitHub organization
(aiftp-tools), and any official communication channel — and also
applies when an individual is officially representing the project
in public spaces.

## Enforcement

If you experience or witness behavior that violates this Code,
please contact the maintainer privately:

- **Email** (preferred): the email address listed on the aiftp-tools
  organization profile at <https://github.com/aiftp-tools>. Subject
  prefix `[conduct]` so it can be triaged separately from technical
  support
- If GitHub Security Advisory is the only channel you have access
  to, you may use it as a last resort with `[conduct, not security]`
  prefixed in the title — but please prefer email, since Security
  Advisories are reserved for vulnerability reports

(Note: keeping conduct channels distinct from security channels is
important so neither category gets buried under the other.)

Reports will be reviewed and investigated promptly and fairly.
The maintainer is obligated to respect the privacy and security of
the reporter of any incident.

## Enforcement actions

Depending on severity and pattern, responses may include:

1. **Private warning** with an explanation of why the behavior was
   inappropriate
2. **Public correction** in the relevant thread, when necessary for
   the community to understand the boundary
3. **Temporary ban** from project spaces for a specified period
4. **Permanent ban** from project spaces

Decisions are at the sole discretion of the maintainer. Appeals can
be sent via the same channels used for the initial report.

## 行動規範（日本語版）

### 私たちの誓い

メンテナと貢献者は、aiftp を **誰にとっても安全で歓迎される場**にすることを誓います。年齢・身体・障害・民族・性別・経験・国籍・宗教・性的指向などに関係なく、すべての参加者が安心して関われる環境を維持します。

### 期待される行動

- Issue / PR / Discussions での **思いやりと敬意**
- 技術的な意見の相違は歓迎、ただし **人格攻撃は不可**
- 言語の壁を意識して **善意で読み取る**（多くの参加者にとって英語は第二言語）
- 議論を遮らず、Off-topic を持ち込まない
- Issue / PR を閉じるときは **理由を簡潔に説明**

### 受け入れられない行動

- ハラスメント、侮辱、人格攻撃
- 不適切な性的な注目・行為
- 他者の個人情報（氏名・所属・認証情報など）を本人の同意なく公開すること
- 技術議論を繰り返し脱線させる
- その他、プロフェッショナルな場として不適切と判断される行動

### 報告窓口

違反を経験・目撃した場合は、メンテナへ非公開で報告してください：

- **メール（推奨）**: aiftp-tools org プロフィールに記載のメールアドレス。件名に `[conduct]` プレフィックスを付けてください
- メール窓口が使えない場合のみ、最終手段として GitHub Security Advisory に `[conduct, not security]` プレフィックスで報告可能（ただし Security Advisory は脆弱性報告専用なのでメールを優先）

すべての報告は迅速・公正に対応し、報告者のプライバシーを保護します。

### 対応

- 私的な警告
- 公の場での訂正（コミュニティが境界を理解する必要がある場合）
- 期間限定のアクセス制限
- 永久アクセス禁止

判断はメンテナの裁量によります。同じ報告チャネルで上訴可能です。
