# aiftp

aiftp は、AI が書いたコードを FTP/FTPS/SFTP サーバへ安全に自動デプロイする開発者ツール。日本の SSH 非対応レンタルサーバでの制作ワークフローを第一の対象とする。

## Language

**制作者 (Builder)**:
aiftp の楔（wedge）となる対象ユーザ。SSH 非対応の日本のレンタルサーバ（ロリポップ／さくら／エックスサーバ等）へ WordPress・静的サイトを納品し、AI エージェント（Claude Code / Cursor 等）でコードを書く、個人または小規模制作スタジオ。aiftp はこのペルソナを最優先で設計する。
_避ける_: 開発者（広すぎる）／ユーザー（曖昧）／非エンジニア（Builder は半技術者であり真の非エンジニアではない）／エンジニア（VPS・SSH を持つ層を含意し、彼らは明示的に対象外）

**Free tier**:
MIT ライセンスで配布される aiftp 本体（CLI + MCP）。安全プリミティブ（二段階ゲート・暗号化バックアップ・ハード除外・Keychain・本番ゲート等）はすべてここに含まれ、有料化しない。
_避ける_: 無料版（口語可だが、ドキュメントでは Free tier に統一）

**Pro tier**:
将来構想中の独自ライセンス版。需要が実証されるまで着手しない（据え置き）。仮に出す場合は破壊的変更を許容するメジャーバージョンで対応し、Free に拡張シーム（プラグイン機構）を先行して仕込まない。
_避ける_: 有料版（協働・利便の提供を指す。安全機能の有料化を意味しない）

**安全プリミティブ (Safety primitives)**:
aiftp を aiftp たらしめる中核機能群。二段階 MCP ゲート（prepare → confirm）、暗号化ローカルバックアップ、ハード除外リスト、OS Keychain、本番プッシュゲート等。Free に永続的に残す。
_避ける_: 安全機能（同義で可だが、"foot-gun を作らない" 設計意図とセットで語る）

## 例: 制作者と開発者の会話

> **制作者**: 「クライアントの WordPress サイト、ロリポップに上げたいんだけど FileZilla で wp-config 上書きしそうで怖い」
> **開発者**: 「それ、まさに aiftp の Builder ユースケース。ハード除外で wp-config は触らないし、上げる前に二段階ゲートで差分を確認できる」
> **制作者**: 「Cursor で書いたコードをそのまま上げられる?」
> **開発者**: 「MCP 経由で AI に上げさせられる。ただし最終確認（confirm）は人間が押す。そこは安全プリミティブだから Free のまま」
